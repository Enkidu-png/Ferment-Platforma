# Requirements: Ferment Platforma — Supabase Migration

**Defined:** 2026-02-24
**Core Value:** The marketplace works exactly as before — artists manage their shops, buyers browse and buy — but the backend is Supabase, making the codebase maintainable with AI assistance.

## v1 Requirements

### Foundation

- [x] **FOUN-01**: Supabase project connected with database, auth, and storage configured in environment *(Phase 1 — doc gap, functionally confirmed)*
- [x] **FOUN-02**: All 8 PostgreSQL tables created (users, products, categories, tags, tenants, orders, reviews, media) *(Phase 1 — confirmed by Phase 4 22/22 table usage)*
- [x] **FOUN-03**: Row Level Security policies implement tenant isolation (products and media scoped to owning tenant) *(Phase 1 — confirmed by smoke tests)*
- [x] **FOUN-04**: RLS anonymous-read policies allow unauthenticated buyers to browse products and storefronts *(Phase 1 — confirmed by smoke tests)*
- [x] **FOUN-05**: JWT custom claims hook embeds `tenant_id` and `app_role` into every auth token *(Phase 1 — confirmed by Phase 6 adminProcedure working in production)*
- [x] **FOUN-06**: Four Supabase client factories created (server component, middleware, client component, service role) *(Phase 1 — confirmed by Phase 4/5/6)*

### Auth

- [x] **AUTH-01**: User can log in with email and password via Supabase Auth *(Phase 2 — confirmed in production via incognito test)*
- [x] **AUTH-02**: User session persists across browser refresh (Supabase SSR cookies) *(Phase 2 — router.refresh() fix deployed and confirmed)*
- [x] **AUTH-03**: Next.js middleware updated to compose Supabase session refresh with subdomain routing as a single Response pipeline *(Phase 2 — working, using x-middleware-rewrite header pattern)*
- [x] **AUTH-04**: tRPC context updated — `ctx.db` replaced with `ctx.supabase`, `protectedProcedure` uses Supabase `getUser()` *(Phase 2 — Phase 4 VERIFICATION 22/22 confirms)*
- [~] **AUTH-05**: All existing artists receive a password reset email so they can log in after migration *(Phase 7 — 2/3 sent; artist1 rate-limited × 2, test account, set via dashboard)*
- [x] **AUTH-06**: New artist can register and create a store (Supabase Auth account + tenants table row, status: pending) *(Phase 2 — /sign-up and /auth/confirm routes confirmed)*

## Out of Scope

| Feature | Reason |
|---------|--------|
| New marketplace features | Migration only — no new functionality until migration is stable |
| Mobile app | Web-first |
| Payments outside Poland | Existing Stripe configuration; expand later |
| Product-level approval | Deferred to v2 — merchant approval covers quality control for now |
| WYSIWYG rich text editor in admin | Complexity not worth it; plain text / markdown is sufficient |
| ORM (Drizzle, Prisma) | Supabase query builder is sufficient; adding ORM increases complexity for non-programmer owner |
| GraphQL | Removed with Payload; not needed with tRPC + Supabase client |
