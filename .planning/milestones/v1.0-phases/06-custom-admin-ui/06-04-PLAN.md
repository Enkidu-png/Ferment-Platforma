**Part A: Extend categoriesRouter**

Add imports to `src/modules/categories/server/procedures.ts`:
```typescript
import z from "zod";
import { adminProcedure } from "@/trpc/init";
import { supabaseAdmin } from "@/lib/supabase/admin";
```

Add to `categoriesRouter` inside `createTRPCRouter({...})`:

```typescript
adminGetAllCategories: adminProcedure.query(async () => {
  const { data, error } = await supabaseAdmin
    .from("categories")
    .select("id, name, slug, parent_id")
    .order("name", { ascending: true });
  if (error) throw new Error(error.message);
  return data ?? [];
}),

adminCreateCategory: adminProcedure
  .input(z.object({
    name: z.string().min(1).max(100),
    slug: z.string().min(1).max(100),
    parentId: z.string().nullable().optional(),
  }))
  .mutation(async ({ input }) => {
    const { data, error } = await supabaseAdmin
      .from("categories")
      .insert({
        name: input.name,
        slug: input.slug,
        parent_id: input.parentId ?? null,
      })
      .select("id")
      .single();
    if (error) throw new Error(error.message);
    return data;
  }),

adminUpdateCategory: adminProcedure
  .input(z.object({
    id: z.string(),
    name: z.string().min(1).max(100),
    slug: z.string().min(1).max(100),
  }))
  .mutation(async ({ input }) => {
    const { error } = await supabaseAdmin
      .from("categories")
      .update({ name: input.name, slug: input.slug })
      .eq("id", input.id);
    if (error) throw new Error(error.message);
    return { success: true };
  }),

adminDeleteCategory: adminProcedure
  .input(z.object({ id: z.string() }))
  .mutation(async ({ input }) => {
    // Guard: check if any products reference this category
    const { count, error: countError } = await supabaseAdmin
      .from("products")
      .select("id", { count: "exact", head: true })
      .eq("category_id", input.id);

    if (countError) throw new Error(countError.message);

    if ((count ?? 0) > 0) {
      throw new Error(`Cannot delete: ${count} product(s) use this category. Reassign them first.`);
    }

    const { error } = await supabaseAdmin
      .from("categories")
      .delete()
      .eq("id", input.id);
    if (error) throw new Error(error.message);
    return { success: true };
  }),
```

Note: `adminCreateCategory` takes a `slug` field. The UI must generate a slug from the name automatically (lowercase, replace spaces with hyphens) to keep the form simple (user only types a name).

**Step A: CategoriesView**

Create `src/modules/admin/ui/views/categories-view.tsx` — `'use client'` component.

Inline-edit table pattern (Claude's Discretion): each row has always-on text inputs for name and slug. A "Save" button per row submits the update. A "Delete" button per row opens an AlertDialog. An "Add Category" button at the bottom appends a new blank row.

Local state for the "new row":
```typescript
const [newName, setNewName] = useState("");
// Auto-generate slug from name: name.toLowerCase().replace(/\s+/g, '-').replace(/[^a-z0-9-]/g, '')
```

For existing rows, use local state per row for editing:
```typescript
// Use a map: { [id]: { name, slug } } for pending edits
const [edits, setEdits] = useState<Record<string, { name: string; slug: string }>>({});
```

When data loads, initialize edits from fetched rows (use useEffect):
```typescript
useEffect(() => {
  if (categories) {
    setEdits(Object.fromEntries(
      categories.map(c => [c.id, { name: c.name, slug: c.slug }])
    ));
  }
}, [categories]);
```

Data + mutations:
```typescript
const utils = trpc.useUtils();
const { data: categories = [], isLoading } = trpc.categories.adminGetAllCategories.useQuery();

const createMutation = trpc.categories.adminCreateCategory.useMutation({
  onSuccess: () => { utils.categories.adminGetAllCategories.invalidate(); setNewName(""); toast.success("Category created"); },
  onError: (e) => toast.error(e.message),
});
const updateMutation = trpc.categories.adminUpdateCategory.useMutation({
  onSuccess: () => { utils.categories.adminGetAllCategories.invalidate(); toast.success("Category updated"); },
  onError: (e) => toast.error(e.message),
});
const deleteMutation = trpc.categories.adminDeleteCategory.useMutation({
  onSuccess: () => { utils.categories.adminGetAllCategories.invalidate(); toast.success("Category deleted"); },
  onError: (e) => toast.error(e.message),
});
```

Table columns: Name (Input), Slug (Input, auto-filled), Actions (Save + Delete).

For the "Add Category" row at the bottom:
- Input for name, slug auto-derived, "Create" button
- `createMutation.mutate({ name: newName, slug: slugify(newName), parentId: null })`

For each existing row:
- Two inputs (name and slug) bound to `edits[row.id]`
- "Save" calls `updateMutation.mutate({ id: row.id, name: edits[row.id].name, slug: edits[row.id].slug })`
- "Delete" wrapped in AlertDialog: "Are you sure? This cannot be undone."

Slugify helper (inline, not a separate file):
```typescript
function slugify(str: string) {
  return str.toLowerCase().replace(/\s+/g, '-').replace(/[^a-z0-9-]/g, '');
}
```

Note: Only show top-level categories (parent_id = null) in this admin table. Subcategory management is out of scope — the seeded subcategories are fixed for v1.

Filter: `categories.filter(c => c.parent_id === null)` for display.
