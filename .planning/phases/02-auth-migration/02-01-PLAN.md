<task type="auto">
  <name>Task 1: Rewrite src/trpc/init.ts — Supabase context + protectedProcedure</name>
  <files>src/trpc/init.ts</files>
  <action>
    Replace the entire file. New implementation:

    1. Remove ALL Payload imports (`getPayload`, `payload/config`, `next/headers`).
    2. Import `createClient` from `@/lib/supabase/server` and `cache` from `react`.
    3. `createTRPCContext` (wrapped in `cache`): calls `const supabase = await createClient()`, then `const { data: { user } } = await supabase.auth.getUser()`, returns `{ supabase, user }`.
    4. Init tRPC with `initTRPC.context<typeof createTRPCContext>().create({ transformer: superjson })`.
    5. `baseProcedure = t.procedure` — no middleware, no Payload injection.
    6. `protectedProcedure = t.procedure.use(async ({ ctx, next }) => { if (!ctx.user) throw new TRPCError({ code: 'UNAUTHORIZED', message: 'Not authenticated' }); return next({ ctx: { ...ctx, user: ctx.user } }); })`.
    7. Export `createTRPCRouter`, `createCallerFactory`, `baseProcedure`, `protectedProcedure`.

    CRITICAL: Use `getUser()` not `getSession()` — `getSession()` does not validate the JWT server-side and is a security vulnerability.
    CRITICAL: `createClient()` is async — always `await` it.
  </action>
  <verify>npx tsc --noEmit 2>&1 | grep -E "init\.ts|Error" | head -20</verify>
  <done>init.ts exports the five symbols, ctx contains { supabase, user }, no TypeScript errors in this file</done>
</task>

    ```typescript

    });
    ```

    **schemas.ts — rename username field to shopName:**
    Keep the same regex validation from `username` but rename to `shopName`:
    - `username: z.string().min(3)...` becomes `shopName: z.string().min(3).max(63).regex(/^[a-z0-9][a-z0-9-]*[a-z0-9]$/, "Shop name can only contain lowercase letters, numbers and hyphens. It must start and end with a letter or number").refine((val) => !val.includes("--"), "Shop name cannot contain consecutive hyphens").transform((val) => val.toLowerCase())`
    - Keep `loginSchema` unchanged.

    **utils.ts — delete the file:**
    Delete `src/modules/auth/utils.ts` entirely. The `generateAuthCookie` function is Payload-specific and is no longer needed. Supabase SSR handles cookies automatically.

    AUTH-05 note: No existing artists exist at this stage (confirmed in CONTEXT.md). AUTH-05 (password reset emails for existing artists) is complete-by-exemption — will be revisited in Phase 7 if real users are onboarded before that phase.
