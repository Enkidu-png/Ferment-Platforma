**Important:** The existing `protectedProcedure` import may not exist yet — add it only if needed. The existing file uses `baseProcedure` only. Add `adminProcedure` import from `@/trpc/init`.

Add to `tenantsRouter`:

```typescript
adminGetTenants: adminProcedure
  .input(z.object({ status: z.enum(["pending", "active", "rejected"]) }))
  .query(async ({ input }) => {
    const { data, error } = await supabaseAdmin
      .from("tenants")
      .select(
        "id, name, slug, status, created_at, " +
        "description, " +
        "products:products!tenant_id(id, name, image:media!image_id(id, url))"
      )
      .eq("status", input.status)
      .order("created_at", { ascending: false });
    if (error) throw new Error(error.message);
    return (data ?? []) as unknown as AdminTenantRow[];
  }),

adminApproveTenant: adminProcedure
  .input(z.object({ tenantId: z.string() }))
  .mutation(async ({ input }) => {
    const { error } = await supabaseAdmin
      .from("tenants")
      .update({ status: "active" })
      .eq("id", input.tenantId);
    if (error) throw new Error(error.message);
    return { success: true };
  }),

adminRejectTenant: adminProcedure
  .input(z.object({ tenantId: z.string() }))
  .mutation(async ({ input }) => {
    const { error } = await supabaseAdmin
      .from("tenants")
      .update({ status: "rejected" })
      .eq("id", input.tenantId);
    if (error) throw new Error(error.message);
    return { success: true };
  }),

adminUndoTenantDecision: adminProcedure
  .input(z.object({ tenantId: z.string() }))
  .mutation(async ({ input }) => {
    const { error } = await supabaseAdmin
      .from("tenants")
      .update({ status: "pending" })
      .eq("id", input.tenantId);
    if (error) throw new Error(error.message);
    return { success: true };
  }),
```

Add local type at the top of the file (alongside existing types):
```typescript
type AdminTenantRow = {
  id: string;
  name: string;
  slug: string;
  status: string;
  created_at: string;
  description: string | null;
  products: Array<{
    id: string;
    name: string;
    image: { id: string; url: string } | null;
  }>;
};
```

Note: The `products` join in `adminGetTenants` fetches product images for the Tinder carousel. PostgREST does not support per-relationship LIMIT, so all products are returned for each tenant. The UI will display the first few. This is acceptable for a small dataset (pending merchants: 0–50 at most).

After adding all procedures, the file should export `tenantsRouter` as before (unchanged export).

**Step B: MerchantsView component**

Create `src/modules/admin/ui/views/merchants-view.tsx` — a `'use client'` component.

State:
```typescript
const [currentIndex, setCurrentIndex] = useState(0);
const [lastAction, setLastAction] = useState<{ tenantId: string; action: 'approve' | 'reject' } | null>(null);
```

Data fetching:
```typescript
const { data: pendingTenants = [], isLoading: loadingPending } = trpc.tenants.adminGetTenants.useQuery({ status: 'pending' });
const { data: approvedTenants = [], isLoading: loadingApproved } = trpc.tenants.adminGetTenants.useQuery({ status: 'active' });
```

Mutations (all with `onSuccess: () => { utils.tenants.adminGetTenants.invalidate(); toast.success(...) }`):
- `approveMutation` → `trpc.tenants.adminApproveTenant.useMutation`
- `rejectMutation` → `trpc.tenants.adminRejectTenant.useMutation`
- `undoMutation` → `trpc.tenants.adminUndoTenantDecision.useMutation`

Handlers:
```typescript
const currentTenant = pendingTenants[currentIndex];

const handleApprove = () => {
  if (!currentTenant) return;
  approveMutation.mutate({ tenantId: currentTenant.id });
  setLastAction({ tenantId: currentTenant.id, action: 'approve' });
  setCurrentIndex(i => i + 1);
};

const handleReject = () => {
  if (!currentTenant) return;
  rejectMutation.mutate({ tenantId: currentTenant.id });
  setLastAction({ tenantId: currentTenant.id, action: 'reject' });
  setCurrentIndex(i => i + 1);
};

const handleSkip = () => {
  setCurrentIndex(i => i + 1);
};

const handleUndo = () => {
  if (!lastAction) return;
  undoMutation.mutate({ tenantId: lastAction.tenantId });
  setCurrentIndex(i => Math.max(0, i - 1));
  setLastAction(null);
};
```

Pending tab render:
- If `loadingPending`: show "Loading..." text
- If `pendingTenants.length === 0` OR `currentIndex >= pendingTenants.length`: show "No pending merchants" empty state
- Otherwise: show `<MerchantReviewCard>` for `currentTenant`, plus a counter "X of Y merchants remaining"

Approved tab render:
- shadcn Table with columns: Name, Slug, Approved date (created_at as stand-in, since we don't have an approved_at column)
- If empty: "No approved merchants yet"

Layout with shadcn Tabs:
