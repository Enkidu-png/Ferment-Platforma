<context>
The core auth issue was a cookie domain mismatch. The original plan 07-04 only fixed client.ts, but middleware.ts and server.ts had the same hardcoded `.${NEXT_PUBLIC_ROOT_DOMAIN}` domain which caused the browser to silently reject session cookies on vercel.app. The middleware runs on every request and was effectively destroying the session after every successful login. All three files are now fixed and deployed.

The GitHub/Vercel integration was also broken — the `main` branch was being auto-deployed but lacked env vars in the Vercel dashboard. CLI deployment bypasses this issue. The `main` branch has been deleted; going forward all work is on `master` deployed via CLI.
</context>
