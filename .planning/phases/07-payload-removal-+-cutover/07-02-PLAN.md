<task type="checkpoint:human-action" gate="blocking">
  <name>Task 2: Add environment variables to Vercel dashboard</name>
  <what-to-do>
Add all required environment variables to the Vercel project before deploying. Claude cannot write secrets to Vercel — this is the one step that requires human action.

**Go to:** Vercel Dashboard → your project → Settings → Environment Variables

**Add these variables (select "Production" + "Preview" for each):**

| Variable | Type | Where to find it |
|----------|------|-----------------|
| `NEXT_PUBLIC_SUPABASE_URL` | Public | Supabase Dashboard → Settings → API → Project URL |
| `NEXT_PUBLIC_SUPABASE_ANON_KEY` | Public | Supabase Dashboard → Settings → API → anon/public key |
| `SUPABASE_SERVICE_ROLE_KEY` | Secret | Supabase Dashboard → Settings → API → service_role key |
| `SUPABASE_JWT_SECRET` | Secret | Supabase Dashboard → Settings → API → JWT Secret |
| `STRIPE_SECRET_KEY` | Secret | Stripe Dashboard → Developers → API keys → Secret key |
| `STRIPE_WEBHOOK_SECRET` | Secret | Stripe Dashboard → Developers → Webhooks → signing secret |
| `NEXT_PUBLIC_STRIPE_PUBLISHABLE_KEY` | Public | Stripe Dashboard → Developers → API keys → Publishable key |

**Verify:** After adding, run `vercel env ls` in the terminal and confirm all 7 variables appear.

CRITICAL: `NEXT_PUBLIC_*` variables are baked into the build bundle at compile time. They MUST be present in Vercel before the first deployment runs.

<task type="auto">
  <name>Task 3: Deploy to production and run smoke tests</name>
  <files></files>
  <action>
**Step 1 — Deploy to production:**
```bash
vercel --prod
```
Wait for the deployment to complete. The command outputs the deployment URL (e.g., `https://ferment-platforma.vercel.app`).

If the build fails on Vercel:
- Check Vercel deployment logs for the error
- Most common cause: missing env var (see NEXT_PUBLIC_* note in Task 2)
- Second most common: TypeScript error in a file not caught locally

**Step 2 — Run smoke tests against the production URL:**

The Playwright config's `baseURL` needs to point at production. Check `playwright.config.ts` to see how it's configured. Run with the production URL:

```bash
PLAYWRIGHT_BASE_URL=https://ferment-platforma.vercel.app npx playwright test \
  tests/smoke/storefront.spec.ts \
  tests/smoke/auth.spec.ts \
  tests/smoke/admin.spec.ts \
  tests/smoke/storage.spec.ts \
  --reporter=line
```

Replace `https://ferment-platforma.vercel.app` with the actual deployment URL from Step 1.

All four critical flows must pass:
1. Storefront loads — artist subdomain pages render, product images load from Supabase Storage
2. Auth flow — login, session persistence, artist registration
3. Admin panel — `/admin` accessible, merchant/product/order views load
4. Storage — images served from Supabase Storage URLs (no blob.vercel-storage.com)
